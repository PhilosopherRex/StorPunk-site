Introduction

StorPunk is a decentralized storage network combining the reliability of cloud storage with the ethos of cyberpunk. It is designed to provide temporary, configurable data storage while incentivizing node operators through a streamlined, market-driven model. Funded by a sustainable 5-year DAO, StorPunk emphasizes censorship resistance, making it ideal for an era of increasing global content restrictions.

>> Fork of the Autonomi Network for temporary data storage
>> Native non-blockchain token with market-based incentives
>> No permanent data, no network bloat
>> Dual-network client for both StorPunk (temporary) and Autonomi (permanent) data
>> Censorship-resistant design for unrestricted access

While Autonomi Network focuses on permanent data storage, StorPunk creates a complementary solution for temporary, configurable data needs. This fork is not an attempt to replace Autonomiâ€”it's meant to create a new option for the temporary data market, which is larger and well-established. Our dual-network client allows seamless access to both networks, providing free, unified management of temporary and permanent data from a single interface.

Our focus on temporary data also addresses concerns about potential cascade collapse that can affect networks with permanent data storage under certain potential conditions. When using StorPunk, clients set an expiration date for their data, and nodes factor this into their pricing. Once data reaches its expiration date, it's simply discarded and so the network cannot fill with old data that is no longer desired.

Temporary Data Model

Data retention is configurable. This reduces network bloat and storage costs while providing flexibility for different use cases. Clients specify how long their data should be stored, and nodes price their services accordingly.

Market-Based Incentives

StorPunk operates on a pure market model where node operators earn revenue directly from storage fees only, without network-issued token rewards to node operators. This approach aligns operator incentives with actual network utility while maintaining token value through direct economic activity rather than inflationary mechanisms.

Native Token

A dedicated non-blockchain native network token ($PUNK) for all network transactions and operations in the StorPunk ecosystem. Claimed from $PUNK-DAO tokens with hold-to-double rewards for long-term holders.

Currency Exchange

Our native exchange runs directly on the StorPunk network, allowing anyone to trade storage tokens without permission from banks, governments, or corporations. This creates a truly open marketplace where data storage and token exchange remain accessible regardless of geographic restrictions or regulatory changes.
We aim to have similar functionality to Bisq).

Sustainable DAO Funding

A 5-year tiered token sale model with decreasing rates, hold-to-double rewards, and soulbound governance tokens. Ensures long-term development through conservative treasury management and community voting via Snapshot.org.

Censorship Resistance

Built for an age of increasing global censorship, StorPunk's decentralized architecture ensures data remains accessible without central points of control. Perfect for political content, free expression, and unrestricted information flow.

Why StorPunk Forks Autonomi â€“ Plain Talk for Investors

Autonomi and StorPunk aim to serve fundamentally different market sectors:

1. Different Jobs

Autonomi = "forever warehouse" (keeps stuff forever).
StorPunk = "short-term locker" (keeps stuff for days, weeks, months, years).

Autonomi is built like a giant digital vault: once you put data in, the network promises to keep it forever. StorPunk is built like a time-stamped locker: you choose how long the data stays, then it's automatically deleted. Two very different models.

2. Bigger Market

Most data is only useful for a short time. That market is 5Ã— larger than the "keep forever" market.

Industry numbers show that roughly 90% of newly created data is only valuable for a short period (logs, IoT streams, temp files, etc.). That short-term slice is worth about $52 billion today and is growing twice as fast as the long-term archive market. Autonomi isn't chasing that slice; StorPunk is.

3. Frictionless Payments

Autonomi

ERC-20 token + ETH gas fees for every storage purchase.

StorPunk

Native token with nano-transaction-fee for all payments and transfers to deter network manipulation while remaining cheaper than any traditional or crypto payment rail -- and if we can do it without transaction fees then we will, but we won't make promises that we can't keep.

4. Not Competition

We're not fighting Autonomi. We're filling a gap they don't serve.

We're copy-pasting Autonomi's code and then trimming out the "forever" parts. Users who need permanent storage keep using Autonomi; users who need short-term storage move to StorPunk. Same ecosystem, different lanes.

5. Win-Win

Storpunk is complementary and will include Autonomi, so no matter what type of data you want to store - permanent or temporary, you can access both networks for free and if you have the token for the appropriate network, you can store it on either too.

With StorPunk's dual-network client, you get the best of both worlds: free access to temporary storage on StorPunk and permanent storage on Autonomi from one simple interface. No switching apps or extra feesâ€”just seamless data management across networks.

This setup grows the entire ecosystem: Autonomi users gain easy temporary storage options, while StorPunk brings more traffic to both. As an investor, you're backing a project that expands opportunities without competingâ€”true win-win for decentralized storage.

6. Uncensorable Data & Trade

StorPunk's highly decentralized native token and built-in native exchange ensure uncensorable data storage, trade, and exchange for everyone, everywhere. No central authority can block, freeze, or restrict access to your data or transactions.

Our native exchange runs directly on the StorPunk network, allowing anyone to trade storage tokens without permission from banks, governments, or corporations. This creates a truly open marketplace where data storage and token exchange remain accessible regardless of geographic restrictions or regulatory changes.
We aim to have similar functionality to Bisq).

7. Sustainable Economics

StorPunk operates on a pure market model where node operators earn revenue directly from storage fees only, without, network-issued token rewards to node operators. This approach aligns operator incentives with actual network utility while maintaining token value through direct economic activity rather than inflationary mechanisms.

By focusing on direct service payments, StorPunk creates a sustainable economic model where token value reflects actual network usage and demand for storage services, without the need for additional token emissions that could affect long-term value stability.

No dilution, no artificial incentivesâ€”just a storage market that pays for itself.

8. No risk of the "Network Death Spiral aka Cascade Collapse"

Permanent storage networks face a structural economic vulnerability that StorPunk's temporary model avoids entirely.

The Problem: Permanent storage networks promise to keep your data forever for a one-time payment. But they still have ongoing costs (electricity, hardware, bandwidth) for all that old data. Their economic model only works if:

  • New uploads keep growing consistently forever
  • Storage costs keep dropping consistently forever

When either assumption slows past an unknown tipping point, a cascade collapse starts - either:

  • fewer uploads begins to drive higher prices per upload, or
  • higher costs for nodes begins to drive prices higher per upload

Thus begins a vicious cycle wherein uploaders move in ever greater numbers to alternative storage methods, which in turn, drives even higher prices, until the network becomes too expensive to maintain, driving an exodus of nods, and collapses with it's data lost.

StorPunk's Solution: Our temporary storage model sidesteps this entirely. When data expires, the costs disappear too. No accumulated burden from old data. No need for infinite exponential growth. Just a sustainable market where you pay for what you use, when you use it.

General Information

What is StorPunk?

StorPunk is a decentralized storage network designed to provide temporary, configurable data storage with a focus on efficiency and user control. It aims to offer a resilient and censorship-resistant alternative to traditional cloud storage, built on a modified version of the Autonomi Network.

How does StorPunk differ from Autonomi Network?

StorPunk forks the Autonomi Network codebase but introduces significant changes, including:

  • Temporary Data Model: Unlike Autonomi's permanent data storage, StorPunk focuses on temporary data storage with configurable retention periods.
  • No Rewards/Giveaways: StorPunk removes the rewards system, relying solely on node-based incentives (e.g., transaction fees, storage fees).
  • Native Token: StorPunk will have its own native token ($PUNK) for network operations, separate from the $PUNK-DAO governance token.
  • Dual-Network Client: Our client supports both StorPunk and Autonomi networks seamlessly.
Why did StorPunk choose temporary data storage?

The decision to focus on temporary data storage is based on three key factors:

  1. Market Opportunity: There is a larger and well-established market for temporary data storage solutions.
  2. Complementary to Autonomi: StorPunk serves as a complementary project that facilitates the base code maintenance and development.
  3. Network Stability: Concerns exist about potential cascade collapse of permanent data networks. Temporary data mitigates this risk.

In practical terms, clients set an expiration date for their data, and nodes factor this into their pricing. When data reaches its expiration date, it's simply discarded.

DAO and Token Sale

What is the $PUNK-DAO token?

$PUNK-DAO is an ERC-20 token that serves two purposes:

  1. Fundraising: Sold in tiers to raise capital for 5 years of sustainable development.
  2. Permanent Governance: Token holders vote on key decisions via Snapshot.org, maintaining their voice throughout the project's lifetime.
How does the tiered sale work?

Tokens are sold in 5 tiers with decreasing rates per USDT:

  • Tier 1: 16 $PUNK-DAO per USDT
  • Tier 2: 8 $PUNK-DAO per USDT
  • Tier 3: 4 $PUNK-DAO per USDT
  • Tier 4: 2 $PUNK-DAO per USDT
  • Tier 5: 1 $PUNK-DAO per USDT

Each tier has a 7-day sale window and limited supply. Tiers are initiated manually based on treasury needs, not automatically. Unsold tokens are burned when the next tier begins.

What is the hold-to-double reward?

Original purchasers who never transfer their $PUNK-DAO tokens receive 2x $PUNK tokens when claiming on network launch. Secondary buyers (those who bought from others) receive 1x $PUNK tokens. This rewards diamond hands and long-term supporters.

Soulbound Tokens and Governance

What happens to my $PUNK-DAO tokens after claiming $PUNK?

After claiming your $PUNK tokens, your $PUNK-DAO tokens become soulbound (permanently locked to your wallet). This means:

  • You cannot transfer or sell them
  • You keep full voting rights forever
  • Your governance voice is permanent
  • Market confusion is prevented (only unused tokens trade)
How do I vote on proposals?

Voting happens on Snapshot.org (gasless and free):

  1. Visit snapshot.org/#/storpunk
  2. Connect your wallet (MetaMask, etc.)
  3. Browse active proposals
  4. Click "Vote" and select your choice
  5. Sign a message (no gas fees!)

Your voting power equals your $PUNK-DAO token balance, whether the tokens are used or unused.

Why are tokens soulbound after claiming?

Soulbound tokens solve several problems:

  • Market Clarity: Buyers always know they're getting unused tokens with full utility
  • Governance Integrity: Voting power stays with committed participants
  • Technical Simplicity: All logic remains on Ethereum

Treasury and Funding

How is the treasury managed?

Treasury management follows conservative principles:

  • Target: 12-18 month runway always maintained
  • New tiers start when runway reaches 6 months
  • Monthly burn rate reports to community
  • Time-locked multisig (3-of-5) controls

What if a tier doesn't reach its minimum funding?

Each tier has a minimum threshold (25% of target). If not met:

  1. Governance votes on refunds or extension
  2. If refunds approved, a separate contract is deployed
  3. Contributors can claim proportional refunds
  4. Any unsold tokens are burned

1. Executive Summary

StorPunk is a decentralized storage network focused on:

  • censorship resistance,
  • a fully decentralized ecosystem with native token and market,
  • temporary configurable data storage,

It is a fork of Autonomi Network to create a complementary additional ecosystem able to tap both the Autonomi network and StorPunk network for permanent and temporary data simultaneously.

By emphasizing temporary data, StorPunk addresses market gaps, reduces network risks, and provides sustainable incentives through market-driven pricing.

Key Value Propositions:

  • Temporary Storage Model: Configurable data retention prevents bloat and potential cascade failures.
  • Sustainable Funding: 5-year tiered token sale with hold-to-double rewards and soulbound governance.
  • Dual-Network Support: Clients access both StorPunk (temporary) and Autonomi (permanent) networks.
  • Integrated Exchange: Built-in trading for multiple currencies, leveraging MetaMask integration.
  • Market-Driven Economics: Pure supply-demand pricing without artificial rewards.

Target Market:

  • Users needing censorship resistance or otherwise fully decentralized temporary cloud storage alternatives.
  • Developers building apps with ephemeral data requirements.
  • Node operators seeking sustainable incentives.
  • Long-term investors in decentralized infrastructure.

2. Introduction and Vision

2.1 The Problems with Autonomi's vision

While Autonomi provides innovative permanent storage, it faces challenges including potential network bloat, cascade collapse risks, and a focus on a narrower market. StorPunk addresses these by specializing in temporary storage, creating a complementary rather than competitive solution.

2.2 StorPunk's alternative Vision

StorPunk envisions a resilient, user-controlled storage network where data lives only as long as needed. Funded through a fair, long-term DAO model, it builds a sustainable ecosystem that rewards commitment while maintaining flexibility.

2.3 CyberPunk Principles

Inspired by cyberpunk ethos, StorPunk emphasizes decentralization, privacy, and resistance to central control, creating a network where users truly own their data destiny.

Central to this vision is StorPunk's strong censorship resistance, making it an ideal platform in an era where many countries are increasingly censoring political websites and online content. By distributing data across a global network of independent nodes with no central authority, StorPunk ensures that information remains accessible and uncensorable, empowering users to share and store data freely without fear of shutdown or interference.

3. Technical Architecture

3.1 Network Overview

StorPunk forks Autonomi for core functionality while adding temporary data handling and dual-network support. The network consists of storage nodes, clients, and an integrated exchange layer.

StorPunk's dual-network client allows users to seamlessly access and manage data on both StorPunk (for temporary storage) and Autonomi (for permanent storage) networks from a single interface. This provides free, unified access to both temporary and permanent data types, enhancing usability while leveraging the strengths of each network without additional costs or complexity.

3.2 Temporary Data Model

Users specify retention periods; nodes price based on duration and capacity. Expired data is automatically discarded during rotation cycles, preventing bloat.

3.3 Node Architecture

Nodes handle storage, replication, and exchange functions. Market-based pricing adjusts dynamically based on utilization.

3.4 Native Token Integration

$PUNK (native token) powers network operations including storage fees and transactions. $PUNK-DAO (ERC-20) handles governance via Ethereum.

3.5 Decentralized Exchange Layer

Supports trading of WBTC, USDT, PAXG, ETH, ANT, $PUNK, and $PUNK-DAO. Integrates with MetaMask for secure, low-maintenance operations.

4. Token Economics and Governance

4.1 Market-Driven Pricing Model

Storage Pricing Mechanism:

StorPunk adopts Autonomi's proven supply-and-demand pricing mechanism, adapted for temporary data storage:

  • Node-Based Pricing: Each storage node assesses its internal resource availability to calculate pricing
  • Capacity Assessment: Nodes compare their current storage load against maximum capacity
  • Supply-Demand Balance: Pricing reflects the ratio between total available nodes (supply) and total storage requests (demand)

Price Discovery Process:

  1. Client queries network for closest nodes to data address
  2. Nodes provide storage cost quotes based on their capacity utilization
  3. Client selects optimal quote and submits data with payment
  4. Node verifies payment, stores data, and replicates to close group

Retention Period Pricing:

  • Short-term Storage: Lower cost multipliers for temporary data
  • Extended Retention: Graduated pricing for longer storage periods
  • Market-Responsive: Pricing adjusts dynamically based on network utilization and retention requirements

4.2 Token Distribution and Supply

$PUNK-DAO Token (ERC-20):

  • Total Supply: Up to 9.6M (if all tiers sell out)
  • Distribution Mechanism: Tiered sales with decreasing rates
    • Tier 1: 16x $PUNK-DAO per USDT (3.2M supply)
    • Tier 2: 8x $PUNK-DAO per USDT (1.6M supply)
    • Tier 3: 4x $PUNK-DAO per USDT (1.6M supply)
    • Tier 4: 2x $PUNK-DAO per USDT (1.6M supply)
    • Tier 5: 1x $PUNK-DAO per USDT (1.6M supply)
  • Burn Mechanism: Unsold tokens burned between tiers
  • Hold-to-Double Rewards: Original holders receive 2x $PUNK on claim
  • Soulbound After Claim: Tokens become non-transferable, retaining governance rights

$PUNK Native Token:

  • Supply: Minted via claims from $PUNK-DAO holders
  • Utility: Network fees, storage payments, transactions
  • Economics: Market-driven with no artificial inflation

4.3 Governance Structure

  • Platform: Snapshot.org for gasless voting
  • Voting Power: 1 token = 1 vote (pre and post-claim)
  • Multisig: Time-locked 3-of-5 for critical functions
  • Proposals: Tier adjustments, treasury management, upgrades
  • Permanent Rights: Soulbound tokens ensure lifelong participation
  • Transition: Governance remains on Ethereum indefinitely

5. Risk Factors and Mitigations

5.1 Technical Risks

Network Infrastructure:

  • Dependency on underlying Autonomi protocol stability and continued development
  • Potential network splits or consensus failures affecting data availability
  • Node churn impacting data replication and network performance
  • Security vulnerabilities in forked codebase requiring ongoing maintenance

Temporary Data Model:

  • Data expiration mechanisms must function reliably to prevent network bloat
  • Risk of premature data deletion due to system errors
  • Complexity of managing configurable retention periods at scale

Token Integration:

  • Smart contract vulnerabilities in native token implementation
  • Exchange layer security risks and potential exploitation
  • Bridge service centralization (mitigated by public verification)

Mitigations:

  • Comprehensive audits and bug bounties
  • Phased testnet deployments
  • Community-driven code reviews
  • Redundant bridge monitoring

5.2 Economic Risks

Market Dynamics:

  • Token price volatility affecting network economics and user adoption
  • Insufficient node operator incentives leading to reduced network capacity
  • Competition from established storage providers offering similar services

Sustainability Concerns:

  • Reliance on market-driven pricing may not sustain network operations long-term
  • Potential misalignment between storage costs and user willingness to pay
  • Economic model dependency on continuous network growth and adoption

Liquidity Risks:

  • Limited exchange listings potentially restricting token accessibility
  • Low trading volume affecting price discovery and user participation
  • Tiered sale failure to meet minimums (mitigated by refund mechanism)

DAO-Specific Risks:

  • Low tier participation (mitigated by minimum thresholds and refunds)
  • Holder fatigue over 5 years (mitigated by regular updates and rewards)
  • Governance centralization (mitigated by time-locked multisig)

Mitigations:

  • Conservative treasury management with 12-18 month runways
  • Monthly transparency reports
  • Hold-to-double incentives for long-term holders
  • Flexible tier timing

5.3 Competitive Risks

Market Position:

  • Established permanent storage networks with larger user bases and developer ecosystems
  • Traditional cloud providers offering competitive pricing and reliability

Mitigations:

  • Emphasis on dual-network client for ecosystem synergy
  • Focus on hardcore decentralization, well beyond other offerings
